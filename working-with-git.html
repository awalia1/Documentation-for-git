Working with Git

1. Add files to your project

This can be done via the touch command or through the interface as usual

Example: $ touch index.html && touch main.css

2. Add new or modified files to staging

Once we add new files or modify exisiting files we can tell git to track them by adding them to staging for the next commit. (think of commits as checkpoints). This will let git know what files the next checkpoint will consist of.

$ git add < filename >

OR

$ git add .

Your files are now on stage to be included in the next commit/checkpoint

3. Add a commit and commit message

With your changes staged, you can create the checkpoint in your work with the following command.

$ git commit -m "commit message"

This command creates the checkpoint for git! Pretty Simple.

4. Push your code.

Finally, push your changes up to Github with the following command:

$ git push

You should see the following message or similar.

        Counting objects: 3, done.
        Delta compression using up to 4 threads.
        Compressing objects: 100% (2/2), done.
        Writing objects: 100% (3/3), 330 bytes | 330.00 KiB/s, done.
        Total 3 (delta 0), reused 0 (delta 0)
        To https://github.com/awalia1/day5-4.git

If you get an error, you may have Initialized your git repository with $ git init
Or you may have Initialized, but you have not yet modified any files since then using: $ git add < filename > and $ git commit -m "commit message"
